# Cursor Development Rules

When generating code for this project:

- Follow Blender 5.1 API conventions.
- Use Python best practices.
- Keep UI, operators, and geometry separate.
- Do not place modeling logic inside UI panels.
- Prefer reusable functions.
- Keep functions under approximately 50 lines where practical.
- Use descriptive variable names.
- Add docstrings to public functions.
- Avoid duplicate code.
- Preserve the existing project architecture unless instructed otherwise.
- Do not modify unrelated files when implementing a feature.
